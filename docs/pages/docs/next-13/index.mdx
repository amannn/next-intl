# Next.js 13 internationalization (i18n)

With the introduction of [the App Router](https://beta.nextjs.org/docs/routing/fundamentals) (i.e. the `app` directory), Next.js 13 provides a new level of flexibility for React apps and new possibilities when it comes to internationalization (i18n).

There are two major changes that affect the usage of `next-intl`:

1. Next.js has removed their built-in internationalized routing
2. Next.js now supports React Server Components

To fill in the gaps and to leverage the new capabilities, `next-intl` now supports:
