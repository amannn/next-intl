- In async Server Components, call `getTranslations()` from `next-intl/server` to read translated strings.
- Messages stay on the server — no client-side JavaScript is shipped for the translation runtime.
- Pair with `setRequestLocale()` near the top of the component so static rendering picks the right locale.

```tsx app/page.tsx
import {getTranslations, setRequestLocale} from 'next-intl/server';

export default async function Page({
  params
}: {
  params: Promise<{locale: string}>;
}) {
  const {locale} = await params;
  setRequestLocale(locale);

  // !mark
  const t = await getTranslations('ServerDemo');
  return <h1>{t('greeting')}</h1>;
}
```
